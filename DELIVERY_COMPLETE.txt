API ENDPOINTS
================================================================================

GET /api/jobs
  Description: Get all jobs with pagination
  Query: pageNumber, pageSize (1-100)
  Returns: JobSearchResultDto with paginated results

GET /api/jobs/{id}
  Description: Get single job by ID
  Returns: JobRawDto with skills

GET /api/jobs/search
  Description: Search jobs by keyword and/or location
  Query: keyword, location, pageNumber, pageSize
  Returns: JobSearchResultDto with search results

POST /api/admin/trigger-fetch
  Description: Manually trigger job ingestion
  Returns: { success, message, jobsAdded, timestamp }

GET /swagger
  Description: Interactive API documentation
  Browser: https://localhost:7001/swagger

DATABASE SCHEMA
================================================================================

JobsRaw (Table)
  PK: Id (int)
  UNIQUE: Url (string)
  FK: N/A (primary entity)
  Relationships:
    - 1-to-1 with JobsProcessed
    - 1-to-many with JobSkills

JobsProcessed (Table)
  PK: Id (int)
  FK: JobRawId → JobsRaw.Id (CASCADE)
  Relationships: 1-to-1 with JobsRaw

Skills (Table)
  PK: Id (int)
  UNIQUE: Name (string)
  Relationships: 1-to-many with JobSkills

JobSkills (Table - Join)
  PK: Composite (JobRawId, SkillId)
  FK1: JobRawId → JobsRaw.Id (CASCADE)
  FK2: SkillId → Skills.Id (CASCADE)
  CHECK: JobRawId > 0 AND SkillId > 0
